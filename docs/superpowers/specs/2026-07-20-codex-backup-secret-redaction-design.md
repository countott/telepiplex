# Codex 上下文备份秘密信息脱敏设计

## 目标

对 `dist/telepiplex-codex-context-20260720` 备份中的高置信度真实凭据进行脱敏，
把秘密值替换为字面量 `***`，然后重新生成并验证加密迁移包。

本次只处理备份副本，不修改 `~/.codex` 中 Codex 正在使用的原始会话、附件、记忆
或登录状态。

## 范围

需要检查并按格式安全重写：

- Markdown、TXT、CSV、JSON、JSONL、脚本和其他可确认编码的文本文件；
- `raw/` 下三个压缩归档解包后的会话、附件和记忆摘要；
- `external-attachments/` 及附件归档中的图片；仅在 OCR 识别出高置信度凭据时，
  用不透明区域覆盖并显示 `***`；
- 支持安全解析的结构化附件。

以下内容不因本次请求而脱敏：

- 用户名、邮箱、电话号码、IP 地址、主机名和文件路径；
- 线程 ID、UUID、普通哈希、Git 提交和不含凭据的 URL；
- 示例值、占位符、环境变量引用和无法确认用途的高熵字符串。

密钥文件
`/Users/young/.codex/secure-keys/telepiplex-codex-context-20260720.key`
不属于扫描输入，不写入备份，也不显示在日志或报告中。

## 高置信度识别规则

只有满足以下至少一种强证据的值才视为秘密：

1. 位于 Authorization、认证或会话 Cookie、密码、口令、API Key、client secret、
   access token、refresh token、session secret、webhook secret 或 private key 等
   明确敏感字段中，且不是示例或占位符；
2. 位于明确的凭据赋值、命令行参数、HTTP 头、URL 敏感查询参数或带认证信息的连接串中；
3. 符合已知真实凭据前缀或完整结构，例如私钥 PEM 块、Bearer/JWT、常见云平台、
   GitHub、Slack 或 OpenAI 令牌格式；
4. 已由上述规则确认的完整秘密值，在备份其他位置再次出现。

示例、占位符和引用包括但不限于 `example`、`changeme`、`placeholder`、`redacted`、
`***`、`<TOKEN>`、`${TOKEN}` 和明显截断的值。它们不进入精确值传播集合。

检测报告只记录类别、文件和替换次数，不记录原始秘密值。

## 处理架构

处理分为四个独立阶段：

1. **展开与盘点**：把现有明文备份复制到受限权限的临时目录，展开三个原始归档，
   识别附件类型，并建立文件清单。原备份在验证完成前保持不变。
2. **发现与替换**：先从结构化字段和强格式中发现秘密，再对已确认的完整值做全包
   精确传播替换。文本值替换为 `***`；图片只覆盖命中的凭据区域。
3. **重建与复扫**：重建嵌套归档和顶层 ZIP，重新运行同一检测器。任何仍存在的
   高置信度命中、无法安全处理的命中或格式损坏都会使流程失败。
4. **加密与交付**：使用现有独立密钥重新生成 AES-256-CBC + PBKDF2 加密包、
   HMAC 和 SHA-256 校验文件。全部新产物先在临时目录中验证；验证通过后，再逐个
   原子替换旧加密产物。

实现应把规则、格式遍历、文本替换、图片处理和报告生成分开，使每一部分可以单独
测试。秘密值集合只存在于处理进程内存中，不写入调试文件。

## 文件保留与清理

最终保留：

- `telepiplex-codex-context-20260720.zip.aes256`
- 对应 `.hmac`、`.plaintext.sha256` 和 `.sha256`
- 解密工具和解密说明
- 独立保存、权限为 `600` 的密钥文件

最终不保留 `dist/` 下的明文备份目录、明文 ZIP、明文 ZIP 校验文件或含秘密的旧加密
副本。临时工作目录只在全部验证通过后按明确路径删除。Codex 原始数据不删除。

## 错误处理

- 文件解析、归档展开、OCR、格式重写或重新打包失败时立即停止，不替换交付产物；
- 检测到高置信度秘密但无法按原格式安全替换时立即停止并报告文件，不静默忽略；
- 加密、HMAC、SHA-256、ZIP 或回环解密验证失败时保留原交付产物并停止清理；
- 日志和错误信息不得包含检测到的秘密原文。

## 验证标准

完成必须同时满足：

1. 脱敏后全包复扫没有高置信度秘密命中；
2. 123 个会话及其线程元数据仍可解析，可读转录和原始 JSONL 结构有效；
3. 三个嵌套归档与顶层 ZIP 均能完整列出和解压；
4. 附件清单和未修改文件的数量、类型保持一致；被修改图片仍可正常打开；
5. HMAC 和密文 SHA-256 校验通过；
6. 实际解密得到的 ZIP 与加密前 ZIP 逐字节一致，且 ZIP 完整性测试通过；
7. 修改密文一个字节后，解密工具在产生明文前拒绝处理；
8. 最终密钥目录权限为 `700`，密钥文件权限为 `600`，FileVault 保持开启。
