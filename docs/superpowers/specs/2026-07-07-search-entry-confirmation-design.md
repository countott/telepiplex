# Search Entry Confirmation Design

## Goal

`/search` must stop searching raw user text directly. Every search must first resolve to a verified movie or series entry, then either confirm the resolved entry and scope with the user or briefly display a clearly resolved movie link before Prowlarr is queried.

This protects the media library from wrong titles, supports exact episode searches, and avoids wasting Prowlarr searches on unverified or unreleased content.

## Current Problem

The current search chain can fall back to the cleaned original query when Douban metadata is not found. That is too loose. A user input such as a typo, a release-style string, or a very specific episode request may still be sent to Prowlarr without proving which media entry it refers to.

The fallback that fetches a page title and cleans it into a query is also too weak for series links. A metadata link can identify a series, but the user still needs to confirm the intended range: whole series, one season, or one episode.

## Required Behavior

All `/search` inputs must follow this rule:

1. Resolve a verified media entry.
2. Resolve the intended search scope.
3. Confirm the entry and scope with the user, except for a single clearly resolved movie link that can auto-advance after a short display.
4. Generate a Prowlarr query from the confirmed structured data.

Raw user text, cleaned page titles, or AI guesses must not be sent directly to Prowlarr.

## Supported Inputs

- Plain text, including natural-language movie or series requests.
- Douban, IMDb, TVDB, TMDB, and MovieDB links.
- Inputs that include episode scope, such as `S02E05`, `第2季第5集`, `第二季全集`, `season 2`, or similar text.

Magnet links remain outside this chain and should use `/magnet` or `/m`.

## Resolution Pipeline

### 1. Direct Parsing

Parse the raw input into a structured search intent:

- title text
- year, when present
- media type hint, when present
- series scope hint:
  - whole series
  - season
  - episode
- season number
- episode number

This parser only extracts intent. It does not verify a media entry.

### 2. First API Match

Run the parsed title through the existing metadata providers:

- Douban lookup
- TVDB lookup
- IMDb/TMDB/MovieDB link extraction as external evidence, followed by provider lookup where possible

If one or more verified entries are found, build confirmation candidates. For series, also fetch season and episode metadata from TVDB when available so the system can validate existence and air status.

### 3. AI Query Normalization

If both Douban and TVDB fail in the first API pass, call AI only as a query normalizer.

The normalizer may:

- fix likely typos
- remove release words, quality tags, subtitle group names, platform names, and noise
- split title, alias, year, season, and episode scope
- return several lookup candidate strings

The normalizer must not:

- claim a verified entry
- invent external IDs
- invent season or episode counts
- invent air dates
- produce a Prowlarr query

Run Douban and TVDB lookup again with the normalized candidate strings.

### 4. AI Verified-Match Fallback

Only if the second API pass still fails, call AI as a verified-match assistant.

The fallback assistant must return either:

- a candidate with at least one verifiable external ID, such as Douban subject id, TVDB series id, IMDb id, TMDB id, or MovieDB id, or
- a blocking result.

Any AI candidate with an external ID must be verified by the corresponding API before it can be shown to the user. If verification fails, block the search.

If AI cannot provide a verifiable external ID, block with a message asking the user to provide a Douban, TVDB, IMDb, TMDB, or MovieDB link, or a clearer query.

## Confirmation Model

The confirmation UI shows combined "entry + scope" candidates.

There is always one recommended first option. The first option is generated by ranking verified candidates against the parsed user intent. It should be visually labeled as recommended.

### Movie Candidates

Movies can be ambiguous. For example, `蝙蝠侠` can match multiple movies. If the movie cannot be uniquely resolved, show multiple movie candidates.

Candidate content:

- Chinese title, when available
- original or English title, when available
- year
- external source and id
- cover image, when available

For movie links, the system must auto-advance when the link resolves to one clear movie. It should show the identified movie for about one second, then continue to Prowlarr search without requiring a button click.

### Series Candidates

Series candidates must combine the entry and range:

- whole series
- one season
- one episode

Examples:

- `绝命毒师 Breaking Bad (2008) - 全集`
- `绝命毒师 Breaking Bad (2008) - S02 整季`
- `绝命毒师 Breaking Bad (2008) - S02E05`

For series links, do not auto-advance. Even if the link clearly identifies a series, the user must confirm the intended range because a series link usually does not encode whether the user wants the whole series, one season, or one episode.

The confirmation should use cover art when available to reduce wrong confirmations.

## Scope Selection Rules

When a series entry is verified, expose the available range choices through confirmation candidates. If the original input already contains a range, make the matching range the recommended first option.

Allowed range choices:

- whole series, only when this is a sensible query for the provider result
- a specific aired season
- a specific aired episode

Unreleased episodes must not be searchable. If the requested episode or season has not aired, show a blocking message that the content has not aired yet and stop the chain.

Do not silently replace an unreleased requested episode with a nearby aired episode.

## Prowlarr Query Generation

Only confirmed candidates can generate Prowlarr queries.

Movie query:

```text
<title> <year>
```

Whole series query:

```text
<series title>
```

Season query:

```text
<series title> Sxx
```

Episode query:

```text
<series title> SxxEyy
```

The title should prefer the verified English or original title for Prowlarr. Chinese title can remain in metadata for later renaming and display.

## AI Prompt Requirements

Prompts must be strict JSON-output prompts.

The query-normalization prompt must include these rules:

- This step does not verify a media entry.
- Return lookup candidates only.
- Do not create external IDs.
- Do not create air dates.
- Do not create season or episode counts.
- Preserve explicit season and episode intent from the user.
- If the input asks for an episode, return that exact intended range in structured fields.

The verified-match prompt must include these rules:

- A result without a verifiable external ID is not acceptable.
- Do not invent IDs.
- Do not invent release status.
- Do not invent season or episode counts.
- If no verifiable match exists, return `blocked_no_verified_match`.
- If the requested episode is known to be unreleased, return `blocked_unreleased`.
- Never output a Prowlarr query.

## State Machine

Add a search preflight stage before Prowlarr search:

1. `SEARCH_RESOLVE_ENTRY`
2. `SEARCH_CONFIRM_ENTRY_SCOPE`
3. `SEARCH_SELECT_RESULT`
4. `SEARCH_SELECT_SUB_CATEGORY`

The existing Prowlarr result selection and save-directory selection can remain after the confirmation stage.

## Error Handling

Block with a clear user-facing message when:

- no verified media entry is found
- AI cannot provide a verifiable external ID
- external ID verification fails
- requested series range does not exist
- requested episode or season has not aired
- provider APIs are unavailable and no verified cached result exists

The message should ask for a clearer title or a supported metadata link when the problem is missing verification.

## Metadata Passed To Download

After confirmation, preserve structured metadata for the download and rename chain:

- media type
- confirmed title
- Chinese title
- original or English title
- year
- external IDs
- selected scope
- season number
- episode number
- evidence source
- cover URL when available

For episode searches, the selected scope should help the later TVDB/AI rename flow map the downloaded file to the intended episode.

## Tests

Add tests for:

- plain ambiguous movie query shows multiple movie confirmations
- plain typo query fails first API pass, AI normalizes, second API pass succeeds
- AI fallback without external ID blocks
- AI fallback with external ID is verified before confirmation
- series query with `S02E05` recommends the exact episode candidate
- series query exposes whole series, season, and episode range choices
- unreleased episode blocks and never calls Prowlarr
- series link requires confirmation instead of auto-advance
- movie link shows a transient confirmation and then searches
- confirmed episode generates `<title> SxxEyy`
- confirmed season generates `<title> Sxx`
- raw query is never sent to Prowlarr when no verified entry exists

## Out Of Scope

- Screenshot OCR input.
- Magnet handling changes.
- Automatic subscription or monitoring for unreleased episodes.
- Changing the post-download media organization strategy beyond passing richer confirmed metadata.
