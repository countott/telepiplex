# Prowlarr 实时搜索计时设计

## 目标

将 Prowlarr 搜索进度从“每 30 秒新增一条提示”改为“在同一条状态消息中从 0 秒开始逐秒计时”，并在搜索结束时立即停止计时。

本次只在本地迭代和验证，不推送远端。

## 交互行为

1. Prowlarr 搜索开始时立即发送状态消息，显示查询词和“已等待 0 秒”。
2. 搜索任务继续在后台线程执行。
3. 搜索未结束时，每秒编辑同一条状态消息，以实际经过时间显示 1 秒、2 秒等进度。
4. 搜索任务结束后立即停止计时，并将状态消息编辑为“搜索完成，用时 X 秒”。
5. 候选排序、索引器摘要、结果键盘、无结果提示和异常提示继续使用现有流程。

## 实现边界

- `_send_search_results` 保留初始状态消息对象，并把它交给 `_search_prowlarr_with_progress`。
- `_search_prowlarr_with_progress` 使用单调时钟计算实际经过时间，不通过固定增量累加，避免事件循环延迟造成计时漂移。
- 每次只编辑原状态消息，不新增周期性进度消息。
- Telegram 状态消息编辑失败时只记录警告，搜索任务继续运行；搜索结果不能因计时展示失败而丢失。
- 不修改 Prowlarr 请求超时、搜索类别、结果排序和下载投递逻辑。

## 测试

- 验证搜索开始时显示 0 秒。
- 验证未完成搜索按秒编辑同一条消息。
- 验证搜索完成后停止更新并返回结果。
- 验证消息编辑发生网络异常时不会取消搜索。
- 运行相关单元测试、Python 编译检查和 `git diff --check`。
