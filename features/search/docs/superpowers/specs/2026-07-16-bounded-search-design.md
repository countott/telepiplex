# Telepiplex 有界媒体搜索与剧集范围确认设计

日期：2026-07-16

状态：已批准，直接进入实现

主责模块：`feature/search`

协作模块：`main`

## 1. 目标

`/s` 只服务于清晰片名、可安全区分的少量候选，以及豆瓣/TVDB直达链接。系统不扩展成影视百科或系列宇宙搜索器；无法在有限步骤内形成可审计结果时，明确拒绝并要求用户补充年份、完整片名、电影/剧集类型，或提供豆瓣/TVDB链接。

核心原则：

- 1–7个合格媒体实体全部展示，由用户选择。
- 超过7个合格实体不截断、不让AI代选，直接拒绝并要求缩小范围。
- 评分只排序和标记推荐，不隐藏已经合格的候选。
- AI只把自然语言整理成待验证意图，不能生成稳定ID、最终媒体契约、Prowlarr查询或Season 00编号。
- 剧集作品身份确认与下载范围确认分成两个状态。
- Prowlarr检索身份与下载后整理身份分离。
- 搜索证据、候选、AI结果、用户范围和整理选择全部即用即弃。
- rename只消费本次下载任务携带的已确认契约，不反查search数据库。

## 2. 输入分类

### 2.1 豆瓣或TVDB链接

- 豆瓣/TVDB作品链接直接锁定链接指向的媒体实体，跳过文本候选竞争。
- 其他来源只补充官方英文标题、日文罗马字、年份、海报和类型，不得把链接换成另一个作品。
- 电影链接确认后进入电影片源搜索。
- 剧集链接进入范围确认。
- TVDB季链接进入“整季/指定单集”。
- TVDB单集链接直接锁定该集，确认后搜索。
- 链接解析失败时直接报错，不降级成普通文本搜索。

Wikipedia继续作为交叉证据，不作为用户直达链接入口。

### 2.2 结构清晰文本

规则层识别：

- 标题；
- 明确年份；
- 明确的电影/剧集类型；
- 全剧、指定季、指定集；
- `Sxx`、`SxxExx`、`1x02`和中文季集表达。

例如：

- `杀死比尔`
- `黑暗荣耀`
- `黑暗荣耀 第一季`
- `黑暗荣耀 S01E03`
- `变形金刚3`
- `蝙蝠侠 1989`

### 2.3 规则无法安全理解文本

只有规则无法确定自然语言含义、首轮无结果或出现非标准范围表达时，才调用一次AI意图解释。AI输出重新进入同一外部来源验证链，不得直接进入Prowlarr。

AI结构：

```json
{
  "status": "parsed|needs_clarification|unsupported",
  "title_hints": ["黑暗荣耀"],
  "media_type_hint": "movie|series|unknown",
  "scope_hint": "work|whole_series|season|episode|latest_aired|unknown",
  "season_number": null,
  "episode_number": 5,
  "numeric_tokens": [
    {"value": 5, "role": "year|official_title_part|season|episode|ambiguous"}
  ],
  "relation_hint": "none|prequel|sequel|special|movie_version|unknown",
  "clarification_reason": ""
}
```

AI不能输出：

- 豆瓣、TVDB、IMDb等稳定ID；
- 用户未提供的年份；
- TVDB季数、集数和播出进度；
- 最终官方英文名或日文罗马字结论；
- Prowlarr查询；
- 保存路径、Season 00映射或`media_metadata`。

## 3. 裸数字规则

- 裸数字不能自动解释为第几部、季或集。
- `变形金刚3`只有在权威来源证明“3”属于官方标题时才通过。
- `蝙蝠侠1`无法证明数字角色时，返回澄清提示，不进入候选展示。
- AI只能标记`ambiguous`或提出`official_title_part`假设，最终必须由来源验证。

## 4. 合格候选与展示门

文本候选至少满足：

- 能形成可审计的媒体实体；
- 至少两路独立来源对齐，或链接/权威稳定ID提供直接锚点并得到补充事实支持；
- 能确定电影或剧集；
- 剧集能锁定TVDB Series ID；
- 标题、年份、类型和海报足以让用户区分；
- 命中来自标题、官方别名或明确分卷关系，不是摘要提及或系列噪声。

展示规则：

- 0个：允许一次AI纠错或意图补全并重新查询；仍为0则拒绝。
- 1个：仍展示海报和身份卡，由用户确认。
- 2–7个：全部展示。
- 超过7个：不截断，要求补充信息或链接。

实体图只负责当前请求内的精确去重。不同年份或不同`movie/series`类型不合并。

## 5. 电影状态机

电影候选卡显示中文名、规范拉丁标题、年份、类型、海报、可验证关系和排序分。

用户选中电影后：

- Prowlarr使用规范英文标题或日文官方罗马字加年份；
- 只搜索电影分类；
- 不询问季集范围；
- 续集、前传和分卷默认作为独立电影实体。

例如：

```text
Kill Bill Vol. 1 2003
```

## 6. 剧集状态机

作品身份确认与范围确认严格分开。

### 6.1 裸剧名

`/s 黑暗荣耀`确认剧集后：

- 单季剧显示“全剧（推荐）/指定集”；
- 多季剧显示“全剧（推荐）/指定季/指定集”。

绝不能从TVDB第一集、`items[0]`或默认值推导`S01E01`。

### 6.2 明确季

`/s 黑暗荣耀 第一季`显示：

- 已完结：总集数、整季（推荐）、指定单集；
- 连载中：计划集数、已播至第几集、搜索当前整季、指定已播单集。

选择指定单集后由用户输入数字，不展开大量集数按钮。输入必须由TVDB校验，不能超过已播范围。

### 6.3 明确集

`/s 黑暗荣耀 S01E01`验证作品、季号、集号和播出状态后，显示一次确认并直接生成：

```text
The Glory S01E01
```

### 6.4 明确全剧

`/s 黑暗荣耀 全剧`验证作品后直接确认：

```text
The Glory 2022
```

### 6.5 数字输入

- 单季剧指定集：只输入集号。
- 多季剧且未指定季：先输入季号，再输入集号。
- 已处于某季上下文：只输入集号。
- Special/Season 00只接受明确链接或权威TVDB事实，不从普通数字猜测。

## 7. 关系验证与交互

关系只在用户选定具体作品后做一次浅层验证，不为所有候选建立复杂关系图。

AI可以提出当前电影可能是某剧的前传、续集、电影版、衍生或特别篇，但来源必须定向验证。

验证出口：

1. 独立电影：显示关系标签，仍按电影搜索和整理。
2. 电影与剧集关系已验证：向用户提供本次整理选择：
   - 归入目标剧集Specials（推荐）；
   - 按独立电影整理；
   - 退出。
3. TVDB存在官方Special编号：整理使用官方Season 00编号。
4. 关系已验证但TVDB无官方Special：本次任务使用本地`S00E100+`。
5. 只有AI猜测且来源无法验证：不建立映射；允许按独立电影处理，或要求用户提供链接。

整理选择只对本次任务有效，不持久化。

## 8. 检索身份与整理身份

Prowlarr只读取`retrieval`，rename只消费已确认的`identity/placement`。

关联电影示例：

```json
{
  "identity": {
    "chinese_title": "想见你",
    "canonical_title": "Someday or One Day",
    "year": "2022",
    "media_type": "movie",
    "external_ids": {}
  },
  "retrieval": {
    "media_type": "movie",
    "scope": "work",
    "query": "Someday or One Day 2022"
  },
  "placement": {
    "library_type": "series",
    "category_kind": "live_action_series",
    "mapping_kind": "related_movie_special",
    "target_series": {
      "chinese_title": "想见你",
      "canonical_title": "Someday or One Day",
      "external_ids": {}
    },
    "season_number": 0,
    "episode_number": 100,
    "mapping_source": "local_reserved"
  }
}
```

即使整理到Specials，检索仍使用：

```text
Someday or One Day 2022
```

不得使用`Someday or One Day S00E100`搜索资源。

## 9. 即用即弃

删除规范媒体实体注册表：

- 不创建或读取`media_entities.db`；
- 不保留`canonical_entities`和`canonical_relations`；
- 不提供历史实体`resolve_exact`；
- 候选、证据、AI结果、评分、Prowlarr结果、范围和整理选择均不持久化。

download下载任务必须携带本次确认的`media_metadata`，直到rename和Plex交接完成。这是任务状态，不是媒体实体数据库。

`S00E100+`在搜索阶段只做临时预留：

- 取消或提交失败时释放；
- 成功交给download后随任务契约流转；
- 不保存为用户偏好或永久关系。

## 10. 失败与超时

- 链接解析失败：直接报错，不降级成文本。
- 清晰电影查询：某一路证据源失败不必机械阻断，只要剩余证据足够。
- 剧集范围：TVDB为硬依赖；不可用时不能进入季集搜索。
- AI不可用：清晰Query和链接正常运行；非标准表达要求改用明确格式。
- 关系验证失败：不影响按独立电影检索。
- 每个阶段独立超时并返回具体错误，不能停留在“正在规划”。
- 海报消息后续必须使用媒体编辑或发送新消息，不能对照片消息调用纯文本编辑。

## 11. 验收案例

- `杀死比尔`：展示两部合格电影，用户选择后按英文名和年份搜索。
- `变形金刚`：1–7个合格实体全部展示；超过7个要求缩小范围。
- `蝙蝠侠1`：拒绝猜测数字。
- `变形金刚3`：来源证明正式标题后正常展示。
- `黑暗荣耀`：确认实体后显示“全剧（推荐）/指定集”。
- 多季剧：显示“全剧/指定季/指定集”。
- `黑暗荣耀第一季`：显示整季或输入单集。
- `黑暗荣耀S01E01`：直接确认单集。
- 连载剧：只允许已播集数。
- 豆瓣剧集链接：锁定剧集后确认范围。
- TVDB单集链接：直接锁定该集。
- 关联电影：始终按电影标题搜索，用户选择独立电影或归入Specials。
- 无官方Special编号：本次任务分配`S00E100+`。
- AI不可用：清晰Query和链接仍可运行。
- 用户选中海报候选后：后台任务和Telegram状态均继续，不出现照片消息`BadRequest`。
